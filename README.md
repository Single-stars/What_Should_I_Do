# 是时候去做点什么了

这是一个通过链接传播的分支式行动测试。用户从“当前身份”进入，根据近期苦恼一步步回答选择题，最后得到一句适合分享的结论：**“是时候去 XXX 了！”**，以及今天就能执行的具体行动。

## 总体思路

- 产品不是传统人格测试，而是“行动型困境测试”。
- 传播核心不是“我是某种人格”，而是“我现在是时候去做某件事了”。
- 测试逻辑采用“身份 + 苦恼 + 卡点 + 限制 + 行动强度”的分支式判断。
- 结果页必须给出具体动作，避免只做情绪安慰或泛泛建议。
- 当前已迁移为 Next.js App Router + TypeScript 项目，便于后续扩展分享图、URL 参数、埋点和服务端能力。

## 关键决策

- 目标用户：所有处于迷茫、转向、职业早期、空窗或探索阶段的年轻人。
- 当前形态：Next.js 单页测验体验，不依赖后端。
- 入口方式：先选择当前身份，再根据身份调整后续问题文案。
- 推荐方式：不是固定总分问卷，而是每个选项给若干结果类型加权。
- 结果表达：统一使用“是时候去 XXX 了！”句式。
- 建议风格：直接行动派，少讲道理，多给下一步。
- 风险边界：不做严肃心理测评，不声称能解决心理疾病、抑郁或严重焦虑。
- 文档规则：每次修改代码或产品逻辑，都必须同步更新 `README.md` 和 `PROJECT_BACKGROUND.md`。

## 本地运行

```bash
npm install
npm run dev
```

常用命令：

- `npm run dev`：启动本地开发服务。
- `npm run build`：执行生产构建检查。
- `npm run start`：运行生产构建后的服务。
- `deploy-to-vercel.cmd`：在 Windows 上一键安装依赖、构建并部署到 Vercel；脚本会优先使用项目内的新版 Node.js，避免系统旧版本 Node 导致构建失败。

## 文件说明

- `app/page.tsx`：客户端测验页面，包括入口页、答题页、结果页和分享卡片。
- `app/layout.tsx`：Next.js 根布局和页面元信息。
- `app/globals.css`：视觉样式、响应式布局、按钮、进度条、结果卡片。
- `lib/quiz-data.ts`：题目数据、选项权重和结果文案。
- `lib/quiz-result.ts`：基于答题历史和答案计算结果的纯函数。
- `lib/quiz-types.ts`：测验核心 TypeScript 类型。
- `package.json`：Next.js、React、TypeScript 依赖和运行命令。
- `deploy-to-vercel.cmd` / `deploy-to-vercel.ps1`：Windows 部署助手，用于生成可分享的 Vercel 公网链接。
- `README.md`：当前状态、关键决策、已完成部分、待办事项和修改记录。
- `PROJECT_BACKGROUND.md`：长期项目背景、总体方针、产品边界和迭代原则。
