# 项目背景

## 项目一句话

做一个容易通过链接传播的行动测试，帮助年轻人从迷茫状态里定位当前卡点，并得到一个今天就能执行的具体建议。

## 用户与场景

目标用户是处于迷茫、转向、职业早期、空窗、毕业前后或自由探索阶段的年轻人。

典型使用场景：

- 朋友或社群里看到链接后点开测试。
- 用户花 3 分钟回答一组选择题。
- 用户得到一句“是时候去 XXX 了！”的结果。
- 用户截图、复制文案或转发链接。
- 用户根据结果完成当天第一个具体动作。

## 总体方针

- 先做传播型 MVP，再根据真实用户反馈提高准确性。
- 先让用户愿意完成和分享，再追求复杂算法。
- 结果必须具体到行动，不能只输出人格描述。
- 题目必须从真实处境出发，避免抽象价值观测试。
- 语言要直接、清楚、有推动感，但不能羞辱或制造焦虑。
- 产品不能宣称心理诊断能力，也不处理严重心理健康问题。

## 产品原则

- 入口要轻：不要用长篇说明阻碍开始。
- 问题要具体：选项描述应像真实生活状态，而不是概念名词。
- 分支要有目的：每一步都要缩小问题空间。
- 结果要可传播：主标题适合截图和转发。
- 建议要可执行：用户看完必须知道今天第一步做什么。
- 隐私要克制：分享卡片只展示结论、类型和短解释，不暴露过多私人信息。

## 关键产品决策

- 传播句式：`是时候去 XXX 了！`
- 首版身份入口：
  - 在校学生
  - 应届/准毕业生
  - 职场新人
  - 工作几年但想转向
  - 待业/空窗/休整中
  - 自由职业/创业探索中
- 首版近期苦恼：
  - 不知道未来做什么
  - 现在做的事没意义
  - 想改变但不知道从哪开始
  - 有很多选择但不敢定
  - 被家人、同龄人或社会期待推着走
  - 每天很累，什么都不想做
- 首版推荐逻辑：选项加权，不做固定总分。
- 首版结果数量：10 类行动处方。
- 首版技术路线：纯静态网页，便于直接打开、部署和传播。

## 当前结果类型

- 信息缺口型：是时候去找一个真实的人聊聊了！
- 选择过载型：是时候把选择缩到 3 个以内了！
- 外界期待型：是时候从别人的期待里退出来了！
- 低能量停滞型：是时候停下无效努力，先恢复能量了！
- 完美主义卡死型：是时候去试一次低成本转向了！
- 路径断裂型：是时候给自己搭一条 7 天过渡路了！
- 反馈缺失型：是时候把想法拿出去见光了！
- 逃避决策型：是时候给这个选择设一个期限了！
- 身份转换焦虑型：是时候先完成一个小闭环了！
- 无意义感型：是时候回到真实问题和真实人了！

## 测试标准

首轮内容测试建议找 20-30 个年轻人，重点观察：

- 是否愿意点开链接。
- 是否能在 3 分钟内完成。
- 是否觉得结果“说中了我”。
- 是否愿意截图或转发结果。
- 是否知道今天第一步该做什么。

MVP 成功参考线：

- 至少 40% 用户愿意截图或转发结果。
- 至少 60% 用户认为结果说中了当前状态。
- 至少 50% 用户能说出今天要做的第一步。

## 技术方针

- 当前使用 Next.js App Router + TypeScript。
- 页面体验仍保持单页测验，不依赖后端。
- 核心内容数据集中在 `lib/quiz-data.ts`，方便快速修改题目、选项和结果。
- 结果计算集中在 `lib/quiz-result.ts`，保持为纯函数，便于后续测试和复用。
- 样式集中在 `app/globals.css`，延续当前视觉和响应式布局。
- Windows 部署通过 `deploy-to-vercel.cmd` / `deploy-to-vercel.ps1` 引导完成，脚本使用项目内新版 Node.js，避免系统旧 Node 版本影响构建。

## 文档维护规则

每次迭代都必须更新两个文档：

1. `README.md`
   - 面向当前项目状态。
   - 记录已完成部分、待办事项、关键决策和重要修改记录。

2. `PROJECT_BACKGROUND.md`
   - 面向长期方向。
   - 记录项目背景、总体方针、产品原则、关键产品决策和技术方针。

如果一次修改改变了产品方向、题目结构、结果类型、传播策略、技术路线或测试标准，必须同步更新这两个文档。

## Git 使用方针

- 建议尽快初始化 Git 仓库。
- 每次迭代前先查看当前状态。
- 每次迭代后提交一次清晰的 commit。
- 不要把多个大方向混在同一个 commit 里。
- 文档更新和对应代码修改应该放在同一次 commit，保证下次加载时能理解当时为什么这么改。

推荐初始命令：

```bash
git init
git add .
git commit -m "Initial action test MVP"
```

推荐日常命令：

```bash
git status
git add .
git commit -m "Update quiz copy and project docs"
```
